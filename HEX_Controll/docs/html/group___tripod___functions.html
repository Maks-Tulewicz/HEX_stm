void printTripodConfig ( void  )

Wyświetl aktualną konfigurację tripod gait.

Funkcja diagnostyczna wypisująca kompletne informacje o aktualnej konfiguracji algorytmu tripod gait. Przydatna do debugowania i weryfikacji ustawień.

Wyświetlane informacje:

  • Wszystkie parametry konfiguracyjne
  • Szacowany czas wykonania cyklu
  • Teoretyczna prędkość robota
  • Grupa nóg i offsety bioder
  • Status algorytmu (ultra szybki/normalny)
Note
Funkcja tylko wyświetla - nie modyfikuje konfiguracji
Output kierowany przez printf() (UART2 w tym projekcie)
// Sprawdzenie konfiguracji przed testem
printf("=== AKTUALNY SETUP ===\n");
printTripodConfig();
printf("=====================\n");

Ustaw parametry tripod gait w runtime.

Funkcja konfiguracyjna pozwalająca na dostrajanie wszystkich parametrów algorytmu bez rekompilacji kodu. Nowe ustawienia wchodzą w życie natychmiast dla kolejnych wywołań tripodGaitCycle().

Walidacja parametrów:

  • step_length: 1.0-10.0 cm (ograniczenia kinematyczne)
  • lift_height: 1.0-8.0 cm (clearance i stabilność)
  • duration: 10-1000 ms (fizyczne ograniczenia serw)
  • points: 5-500 (kompromis wydajność/płynność)

Efekty zmian:

  • ↑ step_length: Większe kroki = szybsze przemieszczanie
  • ↑ lift_height: Wyższa clearance = bezpieczniej na nierównym terenie
  • ↓ duration: Szybsze ruchy = mniej stabilne
  • ↑ points: Płynniejsze ruchy = wolniejsze wykonanie
Parameters

[in]lift_heightWysokość podniesienia [cm]

[in]stance_durationCzas stance phase [ms]
[in]swing_pointsPunkty interpolacji swing

Note
Zmiany są globalne i dotyczą wszystkich kolejnych cykli
Funkcja automatycznie loguje nową konfigurację
Warning
Ekstremalne wartości mogą prowadzić do niestabilności
// Konfiguracja dla szybkiego ruchu

Wykonaj jeden pełny cykl tripod gait.

Główna funkcja algorytmu tripod gait. Wykonuje jeden kompletny cykl składający się z dwóch faz:

Faza 1 (150ms + interpolacja):

  • Grupa A (1,4,5): SWING - ruch nóg w powietrzu
  • Grupa B (2,3,6): STANCE - podtrzymywanie robota na ziemi

Faza 2 (150ms + interpolacja):

  • Grupa A (1,4,5): STANCE - podtrzymywanie robota na ziemi
  • Grupa B (2,3,6): SWING - ruch nóg w powietrzu

Algorytm SWING (noga w powietrzu):

  1. Podniesienie nogi z pozycji tylnej
  2. Łukowa trajektoria nad ziemią (lift_height)
  3. Opuszczenie nogi w pozycji przedniej

Algorytm STANCE (noga na ziemi):

  1. Płynne przesuwanie nogi po ziemi
  2. Z pozycji przedniej do pozycji tylnej
  3. Synchronicznie z całą grupą

Ultra szybka optymalizacja:

  • Używa 30 punktów zamiast 150 (5x szybciej)
  • Brak HAL_Delay() w pętlach interpolacji
  • Całkowity czas cyklu: ~120ms zamiast 300ms
Parameters
[in]pca1Wskaźnik na kontroler PCA9685 lewych nóg (I2C1) lub NULL
[in]pca2Wskaźnik na kontroler PCA9685 prawych nóg (I2C2) lub NULL
[in]directionKierunek ruchu (TRIPOD_FORWARD, TRIPOD_BACKWARD, etc.)
Returns
true Cykl wykonany pomyślnie
false Błąd podczas wykonywania (problemy I2C, kinematyka)
Note
Funkcja automatycznie sprawdza czy kontrolery są dostępne
W przypadku NULL dla pca1/pca2 - odpowiednie nogi są pomijane
Warning
Upewnij się, że robot jest w pozycji stojącej przed pierwszym wywołaniem
// Jeden cykl tripod do przodu
if (!tripodGaitCycle(&pca1, &pca2, TRIPOD_FORWARD)) {
printf("Błąd wykonania cyklu tripod!\n");
}
// Obrót w lewo (w miejscu)

Wykonaj ciągłe chodzenie tripod gait (wiele cykli)

Funkcja wysokiego poziomu wykonująca określoną liczbę cykli tripod gait z automatyczną obsługą błędów i pauzami między cyklami.

Sekwencja działania:

  1. Wyświetlenie informacji o konfiguracji
  2. Sprawdzenie statusu kontrolerów I2C
  3. Wykonanie num_cycles cykli tripod
  4. Pauza 100ms między cyklami (dla stabilności)
  5. Raportowanie czasu wykonania

Obsługa błędów:

  • Stop przy pierwszym błędzie cyklu
  • Szczegółowe logi o przyczynie błędu
  • Bezpieczne zatrzymanie w ostatniej pozycji
Parameters
[in]pca1Wskaźnik na kontroler PCA9685 lewych nóg (I2C1)
[in]pca2Wskaźnik na kontroler PCA9685 prawych nóg (I2C2)
[in]directionKierunek ruchu przez wszystkie cykle
[in]num_cyclesLiczba cykli do wykonania (1-100 zalecane)
Returns
true Wszystkie cykle wykonane pomyślnie
false Błąd w którymś z cykli (szczegóły w logach)
Note
Funkcja blokuje wykonanie na czas: num_cycles × (~120ms + 100ms)
Dla num_cycles=5: ~1.1 sekundy całkowity czas
Warning
Duża liczba cykli może prowadzić do przegrzania serw
// Krótki spacer do przodu (5 cykli = ~1m)
tripodGaitWalk(&pca1, &pca2, TRIPOD_FORWARD, 5);
// Długi spacer (20 cykli = ~4m)
if (!tripodGaitWalk(&pca1, &pca2, TRIPOD_FORWARD, 20)) {
printf("Spacer przerwany - sprawdź hardware\n");
}
// Obrót o ~180° (8 cykli)
tripodGaitWalk(&pca1, &pca2, TRIPOD_TURN_LEFT, 8);
